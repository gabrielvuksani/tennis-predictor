# UI Redesign: Analytical Trading Dashboard — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the current vertical card list UI with a Bloomberg-style trading dashboard (sidebar match list + detail panel) with full mobile responsive support and micro-animations.

**Architecture:** Complete rewrite of `_write_html()`, `_write_css()`, and `_write_js()` inside `src/tennis_predictor/web/generate.py`. The public API (`generate_site()`) and data contract (`predictions.json`) stay identical. No framework migration — all vanilla HTML/CSS/JS output as Python string literals.

**Tech Stack:** Python 3.11+ (generator), vanilla HTML5/CSS3/JS (output), Inter + JetBrains Mono (Google Fonts), Canvas 2D (calibration chart)

**Spec:** `docs/superpowers/specs/2026-03-24-ui-redesign-design.md`

**Security note:** All user-facing data is escaped via `esc()` helper (HTML entity encoding) before insertion into the DOM. The data source is a trusted local `predictions.json` generated by the Python pipeline — not user-supplied input. XSS risk is minimal but escaping is maintained as defense-in-depth.
